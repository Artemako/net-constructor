## Сборка документации

Исходники документации — Markdown-файлы в каталоге `docs/`; навигация и тема задаются в `mkdocs.yml` в корне проекта.

Установите зависимости для документации:

```bash
pip install -r requirements-docs.txt
```

Для предпросмотра в браузере:

```bash
mkdocs serve
```

Откройте адрес из вывода (обычно `http://127.0.0.1:8000`).

Для статической сборки в каталог `site/`:

```bash
mkdocs build
```

При необходимости после сборки можно запустить постобработку HTML для форматирования страниц (если в проекте есть скрипт, например `python build/format_docs_html.py`).
